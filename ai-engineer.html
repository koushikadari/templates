Project 01 · 2024 — 2025 · Northbeam

Neo — a typed ledger for the attribution stack

Replaced a fragile cron-and-Snowflake pipeline with a typed, append-only event ledger backed by Postgres and ClickHouse. Drove p95 attribution latency from 11 minutes to under 40 seconds and made the on-call experience considerably less exciting.

Jan 2024 — Mar 2025 · Tech lead · 4 engineers
Live demo ↗ GitHub ↗
Architecture diagram
Replay UI
Latency profile

Responsibilities

  • Owned the data model and migration plan from a denormalised warehouse schema to an event-sourced ledger.
  • Designed the replay engine that rebuilds materialised views on demand, in roughly seven minutes for a year of data.
  • Specced the on-call runbook, wrote the postmortem template, and trained four engineers into rotation.
  • Drove the partner integration contracts with three downstream teams.

System design

Append-only events in Postgres, fanned out to ClickHouse via Kafka with idempotent consumers. Materialised views are first-class and can be torn down and rebuilt without coordination.

Project 02 · 2023 — Now · Open source

Quill — a Postgres-first job queue for TypeScript

A small library that turns a single Postgres table into a durable, exactly-once job queue with cron, retries and observability built in. Production-grade with under 4,000 lines of code.

Oct 2023 — ongoing · Sole maintainer
npm ↗ GitHub · 4.2k ★
Logo & mark
Dashboard
Type signatures

Responsibilities

  • Designed the at-least-once semantics around SELECT … FOR UPDATE SKIP LOCKED and a fencing-token lease.
  • Built the visibility layer: queue dashboard, OpenTelemetry traces, and a CLI that replays dead-letter jobs.
  • Maintain the project: issue triage, RFC reviews, releases every other Tuesday.
  • Wrote the documentation and two long-form blog posts that drove most early adoption.

System design

A single jobs table, a leases table, and a cron table. No Redis, no broker. The whole queue is one migration; failure modes match Postgres' own.
